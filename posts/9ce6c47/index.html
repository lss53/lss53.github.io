前言

我一路从Wordpress到typecho，再到现在Hexo，一路走来很是有趣，从对网站相关一窍不通，到现在会使用搜索引擎解决问题，真的很开心！为什么一直在换呢？只能说是为了满足自己的折腾乐趣多多！不多说了，现在来记录一下怎么使用Hexo和Github Page搭建博客吧！

域名申请(购买)

以freenom免费域名为例，只记录一下申请过程中踩过的坑。建议用Microsoft Edge 浏览器。

域名不可用

注册时搜索域名显示不可用，如搜zxcvb123.tk时，其实系统已把域名加入到了你的购物车。另外，域名期限默认是三个月，一定要选择12Months@free。

邮箱不可用

换不同的邮箱，直到能接收注册链接时为止！

Complete Order

点Complete Order无反应

  1. 找到注册页面下方的访问IP，搜索访问IP的地址，访问IP和账号地址信息要同一区域。这样容易通过 Recaptcha。

  2. 安装gooreplacers扩展，完毕后网页右上角右键单击扩展图标扩展选项，重定向选项卡，单击新增，弹窗新增 重定向里，匹配模式处输入www.google.com/recaptcha，匹配类型处输入通配符，目标地址处输入recaptcha.net/recaptcha，最后单击提交按钮。

  • 若有广告扩展插件（如Adblock）请关闭。这样一通操作后就可以完成订购、续期了。
  • 若订单号下面出现提示，中文意思您的某些域由于技术错误而无法注册。这些域已被取消。，注册页面右上角Hello XX → Edit Account Details，修改你的账号信息（定位和IP要同一区域）。再次搜zxcvb123.tk注册。

登录失败

登录时出现Login Details Incorrect. Please try again.提示，不必慌张，在登录页面点Request a Password Reset，输入注册时的邮箱，重置密码即可。

域名DNS解析

  1. 以DNSPOD为例，建议使用微信扫码快速注册，再按提示实名认证。

  2. 登录DNSPOD，管理控制台→DNS解析→我的域名→添加域名→输入zxcvb123.tk→确定。

  3. 登录freenom，Services→My Domains→Manage Domain→Management Tools→Nameservers→Use custom nameservers (enter below) 。填入DNSPOD的DNS服务器地址：peony.dnspod.net和peanuts.dnspod.net， 然后点击change nameservers按钮，你就大功告成了。

写日志

整体流程：创建一篇新文章（new） → 编辑文章→ 产生静态档（generate）→部署到GitHub（deploy）

创建文章

新建一篇文章。文档会在source\_drafts\、source\_posts\或source\下，看你选择什么layout的模板。

Hexo有三种预设layout: draft（草稿）、post（公开文章）和page（页面），命令中没写layout默认是post。

你也可以在scaffolds中编辑或添加你要的模板！

刚新建的文档是Markdown（.md）的格式，这也是Hexo的一大特点！如果对Markdown语法还不是太熟，可以搜索引擎搜索一下。

Vercel Deploy(方法2)

注册Vercel

  • 打开vercel → sign up(注册) 。

  • 单击Continue with GitHub → 新窗口Authorize Vercel - 用户配置 → 输入密码，单击Confirm password → 单击绿色按钮Authorize Vercel授权，完成网页自动关闭。

给Vercel授权

  • 回到Vercel，单击Import Git Repository(导入Git存储库)下面的∨(下拉菜单) → Ass GitHub Org or Account(添加GitHub组织或帐户)。

  • 新窗口Installing Vercel - 用户配置 → Only select repositories下面的Selected repository下拉菜单，选择允许授权的GitHub仓库 → 单击Install，完成后网页自动关闭。

导入仓库

  • 回到Vercel，单击Import Git Repository下面已授权的仓库后的Import蓝色按钮。

  • 单击PERSONAL ACCOUNT(个人帐户)后的Select。

  • FRAMEWORK PRESET(框架预设)选择Hexo。

  • 展开Build and Output Settings，BUILD COMMAND中先单击OVERRIDE再输入hexo clean && hexo g && gulp。

  • 单击Deploy。耐心等待，撒花表示成功，单击Visit访问博客。

  • 以后每次git push后，Vercel自动构建和部署博客。

Vercel Domains

登录vercel，单击你的项目，再依次单击Settings→Domains以后分两步进行。

GitHub Actions(方法3)

什么是 GitHub Actions？

创建仓库

需要在GitHub Pages里创建两个仓库，一个 git push Hexo，另一个部署静态网站（建议命名为your-username.github.io）。

GitHub token

任意页面的右上角，单击您的个人资料照片，Settings → Developer settings → Personal access tokens → Generate new token

  • Note：随便填一个描述名称，如 bash script

  • 权限复选框：全部勾选（只勾选repo也可以）

  • 最后单击 Generate token 等待。复制个人访问令牌，它只显示一次，关闭网页前请保存好，否则只能重新生成。

Giee token

任意页面的右上角，单击您的个人资料照片，个人设置 → 私人令牌 → 生成新令牌

  • 私人令牌描述：随便填一个描述名称，如 bash script

  • 私人令牌所拥有的权限：全部勾选（只勾选project也可以）

  • 最后单击提交，等待。复制好私人访问令牌，本页面关闭后，平台将不再显示私人令牌，请妥善保存。

博客根目录里创建.github/workflows/main.yml文件。

.github创建的命令为mkdir .github。
